0.1.0 (2019-02-06)

Full Changelog

Features:

  • [FEATURE]: Cached includes will only read a partial with the same name and URLSearchParams once. Any partial with the same name, but different URLSearchParams will be read multiple times.
  • [FEATURE]: Helper directives/functions can be registered via Engine.registerHelper and will be compiled along with the built-in directives
  • [FEATURE]: Debugger option sets a standard debugger statement into each rendered template. Metadata accessibility from within templates.
  • [FEATURE]: Include parameters can be set to either URLSearchParams to initiate a read or JSON to add the parameters to the partial scope
  • [FEATURE]: Passing parameters to the include directive for use on the server and/or within the partial template itself using the options.includesParametersName
  • [FEATURE]: options.defaultBaseTemplateName added to allow naming of the template being compiled
  • [FEATURE]: Assignments can be made that are outside the primary context passed into the rendering function (i.e. "it"). Templates can designate something like {{ myVarName = 'Some value' }} which can later be retrieved via {{ myVarName }}. This is a static namespace for use during compilation rather than during rendering (e.g. renderFunction({ myVarName: 'Some value' }) with the template using {{ it.myVarName }}).
